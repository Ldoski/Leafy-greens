# TFLite Micro — Spinach Freshness Classifier

## What This Folder Contains

| File | Purpose |
|---|---|
| `tflm_inference_nir.cpp` | ESP32 benchmark — measures inference accuracy and speed |
| `tflm_model_nir.h` | INT8 quantised TFLite flatbuffer (2,688 bytes) |

The model is generated by `../aifes/train_model_NIR.py` alongside the AIfES weights. Same architecture, same training data — just a different runtime.

---

## The Model

**Architecture:** `Input(10) → Dense(16, ReLU) → Dense(3, Softmax)` — identical to the AIfES model.

**Quantisation:** INT8 (TensorFlow Lite default optimisation). Weights and activations are quantised to 8-bit integers. The flatbuffer is 2,688 bytes compared to 908 bytes of AIfES float32 weights.

**Runtime:** EloquentTinyML v3 + tflm_esp32 library.

---

## How to Flash and Benchmark

In the `ESP32/` folder:
```bash
pio run -e tflm_inference_nir --target upload
```

---

## Benchmark Results (Batch 4 test set, 317 samples, 100 repeats)

| Metric | TFLite Micro | AIfES (float32) |
|---|---|---|
| Overall accuracy | 66.9% | 67.8% |
| Inference time | 304.2 µs | 74.7 µs |
| Model size | 2,688 bytes | 908 bytes (weights only) |

TFLite Micro is ~4× slower than AIfES on the ESP32 because INT8 operations run in software without hardware acceleration. AIfES float32 benefits from CMSIS-optimised dot products. Per-class accuracy is identical across both frameworks, confirming the accuracy difference is in the quantisation, not the data.

The TFLite model is regenerated automatically when you run:

```bash
python train_model_NIR.py
```
This writes `tflm_model_nir.h` directly into this folder.
